## FIX_SUMMARY

Add page-type classification to `audit_published_pages.py`. The classifier
distinguishes `site_index` (the site root) from `paper_page` (a normal paper
reading page) from `manifest` (the `published_pages.json` JSON itself) from
`unknown` (fallback). Paper-level checks (Claims, Glossary, Resolver, essay,
zh-CN UI markers) are skipped on `site_index` and `manifest`; severe checks
(`template_leak`, `raw_dict`, `old_footer`) still fire on every page type
because those would still be real regressions on the root index.

In their place, two new check groups run on the appropriate page types:

- **Index-specific checks** (`_check_site_index`): title present, ≥1
  published-page link, manifest reference, link-vs-manifest delta within
  tolerance.
- **Manifest-specific checks** (`_check_manifest_pages`): JSON valid,
  `pages` list present, every entry has `slug` + `title` + `path`, no
  duplicate slugs / paths.

## PAGE_TYPE_CLASSIFICATION

`_classify_page_type()` in `audit_published_pages.py` uses these rules, in
order:

1. **Manifest**: body parses as JSON containing `pages: [...]` and at least
   one `slug` → `manifest`.
2. **Site-index (explicit root)**: URL is the requested site root
   (`--include-root` flag) → `site_index`.
3. **Site-index (URL match)**: URL path is empty AND URL equals
   `<site_root>` → `site_index`.
4. **Site-index (manifest signals)**: URL path is empty AND body contains
   `published_pages.json` / `Paper Reading Pages` / `Published pages` /
   links to `/<slug>/` paths → `site_index`.
5. **Paper-page (default)**: anything else → `paper_page`.

The fallback for unrecognised URLs is `paper_page` (safe default — full
paper-level check set runs).

## ROOT_INDEX_EXEMPTION

When `page_type == "site_index"`, the audit **does not run** these paper-level
checks (all from `PAPER_LEVEL_CODES` in `audit_published_pages.py`):

- `missing_resolver_trail`
- `missing_claims_section`
- `missing_glossary`
- `no_visible_claim_id`
- `no_evidence_label`
- `glossary_no_explicit_definition`
- `essay_missing_markers`
- `zh_cn_markers_weak`
- `empty_claim_id`

The audit **still runs** these severe checks (`SEVERE_CODES`):

- `template_leak`
- `raw_dict`
- `old_footer`

In their place, the audit runs `_check_site_index()`:

- `index_missing_title` — site_index has no `<title>`
- `index_no_page_links` — site_index has zero published-page links
- `index_links_mismatch` — site_index link coverage drops >50% vs the manifest
- `index_no_manifest_link` — info-only; site_index does not reference
  `published_pages.json`

## MANIFEST_CHECKS

`_check_manifest_pages()` runs once per audit (top-level) and is always
emitted. Codes:

- `manifest_pages_not_list` (error) — `published_pages.json.pages` is not a list
- `manifest_empty` (warning) — `pages` list is empty
- `manifest_incomplete_entries` (warning) — entries missing `slug` / `title` / `path`
- `manifest_duplicate_slugs` (warning) — duplicate slug strings
- `manifest_duplicate_paths` (warning) — duplicate `path` strings (compared
  case-insensitively, trailing slashes stripped)

The manifest check is recorded as a synthetic `_manifest_shape` page entry
(classified as `manifest`) in the report.

With `--include-manifest`, the manifest JSON is also fetched and audited as a
`manifest`-typed page entry (`page_type_override=PAGE_TYPE_MANIFEST`), which
adds an `info`-level `manifest_ok` entry on success.

## LIMITATIONS

- The classification is rule-based and conservative: an URL with an empty
  path that does not match the `--site-root` exactly must contain manifest
  signals (`published_pages.json` / `Paper Reading Pages` / `Published pages`
  / links to `/<slug>/`) to be classified as `site_index`. Otherwise it
  falls back to `paper_page` (the safe default).
- The `_audit_page` function still runs `_check_template_leak` /
  `_check_old_footer` / `_check_raw_dict` on every page type, including
  `site_index`. This is intentional — those would still be real regressions
  on the root index. A root index that contained `{% else %}` would still
  FAIL the audit.
- The `--include-manifest` flag defaults to off. Run with
  `--include-manifest` to also fetch and audit the `published_pages.json`
  JSON itself. The manifest-shape check (no duplicate slugs, etc.) always
  runs and is recorded as a synthetic `_manifest_shape` page entry.
